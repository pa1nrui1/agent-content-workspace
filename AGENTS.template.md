# Agent 工作规则

## 一、工作区定位

本工作区用于管理创作者的内容生产流程，包括选题、写作、视觉制作、发布检查、复盘和长期经验沉淀。

Agent 在本工作区工作时，必须优先读取配置文件、平台规则和 Learning 文件，再进行内容生成、修改、检查或归档。

## 二、读取顺序

执行任何内容任务前，Agent 必须按顺序读取：

1. `config/creator-profile.template.md` 或用户已复制出的 `config/creator-profile.md`
2. `config/content-pillars.template.md` 或用户已复制出的 `config/content-pillars.md`
3. 对应平台规则文件
4. `Learning.md`
5. 相关历史草稿、最终稿、复盘、样本、脚本和预测文件

如果用户尚未完成配置，Agent 应先引导用户完成配置，而不是直接开始生成内容。

## 三、首次使用引导

如果用户刚下载或刚初始化本工作区，Agent 不应直接开始写内容。

Agent 应像聊天一样，一次只问一个问题，并在用户回答后进入下一个问题。不得一次性把所有配置问题都列给用户。

每一步都要做三件事：

1. 只问当前这一个问题。
2. 根据用户回答更新对应配置文件。
3. 简短复述已记录的内容，再进入下一个问题。

按以下顺序引导：

1. 你的创作者身份是什么？
   - 你希望别人怎么认识你？
   - 你有哪些专业背景、经历或可信来源？
   - 哪些身份信息可以公开，哪些不能公开？

2. 你的受众是谁？
   - 你主要想写给谁看？
   - 他们现在最关心什么？
   - 他们已经懂什么，哪些地方需要你讲清楚？

3. 你有什么样的内容想法？
   - 你想长期围绕哪些方向创作？
   - 最想持续讲的 3-5 个主题是什么？
   - 有哪些内容明确不想做？

4. 你准备在哪些平台发布？
   - 哪个平台用来写长内容？
   - 哪个平台用来做图文？
   - 哪个平台用来做视频？
   - 每个平台承担什么作用？

5. 每个平台应该怎么写？
   - 长文平台要更专业完整，还是更故事化？
   - 图文平台要更像作品展示、经验分享，还是教程？
   - 视频平台要更口语、更犀利，还是更稳重？

6. 你喜欢什么文风？
   - 喜欢什么标题、开头和表达节奏？
   - 有没有喜欢的账号、文章、帖子或视频？
   - 如有链接，Agent 可以帮助总结成文风卡。

7. 视觉素材怎么处理？
   - 是否使用真实截图？
   - 是否使用封面图、流程图、图片卡片或视频画面？
   - 哪些素材不能公开？

8. 哪些信息绝对不能公开？
   - 真实姓名、电话、邮箱、客户信息、账号后台、本地路径、未发布草稿、截图、视频、音频等。

9. 是否开启外部样本文风学习？
   - 微信公众号：保存文章。
   - X / Twitter：打开浏览器，必要时等待用户登录，保存公开内容。
   - 小红书：打开浏览器，必要时等待用户登录，保存公开内容和截图。
   - 抖音：下载 MP4，使用 FFmpeg 转音频，再使用 whisper.cpp 或本地转写工具转文字。

如果用户已有明确想法，Agent 应帮助用户整理进配置文件。

如果用户没有明确想法，Agent 应请用户提供同类型优秀账号、文章、视频或主页链接，并帮助用户总结定位、内容主线、平台规则、文风偏好、视觉规则和隐私边界。

## 四、外部样本采集与文风学习

当用户提供微信公众号、X / Twitter、小红书、抖音等链接，并要求“学习文风”“保存到喜欢的文风”“分析表达方式”“提炼钩子”时，Agent 必须进入外部样本采集流程。

执行步骤：

1. 判断链接平台。
2. 调用 `node scripts/capture-source.mjs "<链接>"`。
3. 如果浏览器需要登录，等待用户完成登录。
4. 采集完成后读取生成的文风卡。
5. 询问用户是否写入 `Learning.md` 或 `config/style-preferences.md`。

平台规则：

- 微信公众号：调用 `skills/wechat-article-capture` 保存文章。
- X / Twitter：使用浏览器打开链接，必要时等待用户登录，保存公开内容。
- 小红书：使用浏览器打开链接，必要时等待用户登录，保存公开内容和截图。
- 抖音：不使用 `yt-dlp` 作为默认路径。必须通过浏览器拦截 MP4 媒体请求，下载 MP4，用 FFmpeg 转音频，再用 whisper.cpp 或本地转写工具转文字。

禁止事项：

- 不保存 cookie、账号密码、私信、后台、草稿箱。
- 不把原文、截图、视频、音频、转写全文提交到公开仓库。
- 不直接复制他人原创表达。

## 五、选题前规则

推荐任何选题前，Agent 必须完成：

1. 创作者定位匹配检查。
2. 内容主线匹配检查。
3. 平台适配检查。
4. 历史查重。
5. 风险检查。
6. 差异化判断。

Agent 不得直接凭当前灵感推荐选题。

查重时至少检查：

- 当前平台 `drafts/`
- 当前平台 `final/`
- 当前平台 `retros/`
- 当前平台 `samples/`
- 当前平台 `scripts/`
- 当前平台 `predictions/`
- 其他平台相关内容
- 全局样本、脚本、预测和复盘文件

## 六、写作前规则

写作前必须确认：

1. 目标平台。
2. 内容类型。
3. 当前产物状态。
4. 是否需要事实核验。
5. 是否涉及隐私、版权或专业风险。
6. 是否需要用户补充材料。

如果信息不足以开始写作，Agent 应先向用户说明缺口，并请求补充材料。

## 七、内容钩子规则

图文类内容必须检查：

1. 标题。
2. 首图。
3. 前两行正文。
4. 多图递进。
5. 隐私和版权风险。

视频类内容必须检查：

1. 前三秒口播。
2. 前十秒承接。
3. 节奏推进。
4. 画面辅助。
5. 结尾讨论口。

长文类内容必须检查：

1. 标题。
2. 开篇问题。
3. 结构推进。
4. 依据和边界。
5. 结尾判断。

## 八、视觉制作规则

Agent 不得默认某一种视觉形式适用于所有选题。

制作视觉素材前，必须确认：

1. 内容类型。
2. 目标平台。
3. 用户参考案例。
4. 视觉形式。
5. 是否使用真实截图。
6. 是否使用信息卡、流程图、生成图或 HTML 幻灯片。
7. 是否存在隐私风险。
8. 是否需要第三方 Skill。

## 九、第三方 Skill 使用规则

第三方 Skill 应作为内容生产链条中的专业能力模块使用。

- 内容评分、发布前预测、发布后复盘可使用 `cheat-on-content`。
- 图文图片卡片、知识卡、流程图可使用 `baoyu-skills / baoyu-xhs-images`。
- HTML 幻灯片、视频辅助画面、复杂概念可视化可使用 `frontend-slides`。

调用第三方 Skill 前，Agent 应先判断该 Skill 是否适合当前任务。

同步或修改第三方 Skill 前，必须核对许可证，保留原作者信息和原 LICENSE。

## 十、定稿规则

所有可能发布的内容必须经过用户明确确认。

用户未确认前，只能放入 `drafts/` 或 `中间产物/`，不得放入 `final/` 或 `最终稿/`。

Agent 在内容接近可发布状态时，必须询问：

```text
这是否确认为最终稿？
```

## 十一、Learning 更新规则

复盘后，如出现稳定偏好、用户纠正、有效经验、失败判断或被推翻假设，Agent 应询问用户是否写入 `Learning.md`。

Agent 不得擅自修改长期规则。

写入 Learning 时，应记录：

1. 日期。
2. 经验类型。
3. 具体规则。
4. 适用范围。
5. 来源或证据。

## 十二、隐私规则

Agent 不得写入或公开：

- 真实姓名
- 真实账号
- 电话
- 邮箱
- 地址
- 客户信息
- 案号
- 合同
- 聊天记录
- 平台后台数据
- token
- cookie
- secret
- auth 信息

所有真实信息必须使用占位符。

常用占位符包括：

```text
<创作者名称>
<平台A>
<平台B>
<平台C>
<内容主线A>
<目标受众>
<项目A>
<SkillA>
<隐私信息>
```
