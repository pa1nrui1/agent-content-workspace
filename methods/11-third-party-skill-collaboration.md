# 第三方 Skill 协作方法

## 目标

第三方 Skill 是内容生产链条中的专业能力模块。

本工作区关注的不是如何堆叠更多 Skill，而是在正确环节使用合适 Skill 解决具体问题。

## 内容预测与复盘类 Skill

适用任务：

- 内容评分。
- 发布前预测。
- 发布后复盘。
- 判断偏差记录。
- 规则迭代。

使用目标：

把“我感觉这条会好”变成可记录、可对照、可修正的判断。

不适用任务：

- 替用户决定账号定位。
- 替用户决定所有选题。
- 替用户跳过人工判断。

## 图文图片类 Skill

适用任务：

- 图文封面。
- 信息卡。
- 知识卡。
- 流程图。
- 对比图。
- 多图结构。

使用前必须确认：

- 内容是否适合图片卡片。
- 用户是否确认视觉方向。
- 是否存在隐私风险。
- 是否需要参考案例。

## HTML / Slides 类 Skill

适用任务：

- HTML 幻灯片。
- 视频辅助画面。
- 流程演示。
- 项目结构展示。
- 复杂概念可视化。

使用前必须确认：

- 内容是否需要强视觉表达。
- 平台是否适合演示型画面。
- 是否需要导出图片或视频。
- 是否存在隐私风险。

## 同步规则

同步第三方 Skill 前必须：

1. 核对许可证。
2. 保留原 LICENSE。
3. 保留原作者信息。
4. 在 NOTICE 中记录来源、用途和许可证。
5. 单独说明本地修改。
