# 先配置再生产

## 核心原则

Agent 不应在缺少上下文时直接生产内容。

内容生产的第一步不是写标题、写正文或生成图片，而是配置创作者、平台、内容主线、文风、视觉策略和隐私边界。

## 必须先配置的内容

正式生产内容前，应完成以下配置：

1. 创作者定位。
2. 内容主线。
3. 平台规则。
4. 文风偏好。
5. 视觉规则。
6. 隐私规则。
7. Learning 文件。

## 用户已有清晰想法时

如果用户已经知道自己要做什么内容，Agent 应引导用户直接填写配置文件：

- `config/creator-profile.template.md`
- `config/content-pillars.template.md`
- 对应平台规则文件
- `config/style-preferences.template.md`
- `config/visual-rules.template.md`
- `config/privacy-rules.template.md`

## 用户没有清晰想法时

如果用户尚未形成清晰定位，Agent 应引导用户提供 3-10 个优秀案例链接或文件。

案例可以包括：

- 账号主页。
- 文章。
- 视频。
- 图文笔记。
- 播客。
- 网站。
- 开源项目。

Agent 应从案例中抽象：

1. 内容主线。
2. 目标受众。
3. 标题模式。
4. 开头方式。
5. 结构节奏。
6. 视觉风格。
7. 可以学习的地方。
8. 禁止照搬的地方。

## 输出要求

配置阶段的输出不是成品内容，而是可复用的工作区规则。

配置完成后，Agent 才能进入选题、写作、视觉制作和复盘流程。
