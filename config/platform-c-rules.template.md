# 平台 C 规则

## 一、平台定位

平台 C 用于承载短视频、口播、节奏推进和连续系列内容。

## 二、内容类型

平台 C 主要支持：

1. 口播短视频。
2. 工具或流程展示。
3. 问题拆解。
4. 系列化讲解。
5. 复盘或踩坑记录。

## 三、口播结构

建议结构：

1. 前三秒给出冲突、反差、后果或真实卡点。
2. 前十秒说明为什么值得继续看。
3. 引出具体问题。
4. 拆一个具体点。
5. 展示流程变化。
6. 说明边界。
7. 留下讨论口或下一条线索。

## 四、前三秒规则

前三秒不得使用平铺背景介绍。

不建议开头：

- “大家好，今天分享……”
- “我最近做了一个……”
- “随着某某的发展……”
- “今天给大家介绍一下……”

建议开头：

- 很多人以为问题在 A，但真正麻烦的是 B。
- 这件事最容易出错的地方，不是最后一步，而是第一步。
- 如果前面没弄清楚，后面做得越快，风险越大。

## 五、节奏规则

视频每 15-20 秒应有一次推进。

推进方式包括：

- 原来 vs 现在。
- 很多人以为 vs 实际问题。
- 直接生成 vs 前置流程。
- 工具能做什么 vs 不能做什么。
- 结果展示 vs 边界说明。
