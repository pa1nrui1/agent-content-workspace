# Learning 更新机制

## 目标

Learning 文件用于记录长期稳定规则。

它不是草稿区，也不是发布数据表，而是工作区的长期记忆。

## 可以写入 Learning 的内容

可以写入：

- 用户长期偏好。
- 用户明确纠正。
- 平台钩子经验。
- 视觉策略经验。
- 第三方 Skill 使用经验。
- 验证过的规律。
- 被推翻的判断。
- 待验证假设。

## 不应写入 Learning 的内容

不应写入：

- 临时想法。
- 未确认的草稿。
- 真实隐私信息。
- 平台后台截图。
- 客户资料。
- 未授权样本原文。

## 更新流程

复盘后，Agent 应判断是否产生新经验。

如果产生新经验，Agent 应询问用户：

```text
是否将这条经验写入 Learning？
```

用户确认后，再写入对应栏目。

## 写入格式

每条经验应包含：

1. 日期。
2. 类型。
3. 具体规则。
4. 适用范围。
5. 来源或证据。
