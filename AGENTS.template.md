# Agent 工作规则

## 一、工作区定位

本工作区用于管理创作者的内容生产流程，包括选题、写作、视觉制作、发布检查、复盘和长期经验沉淀。

Agent 在本工作区工作时，必须优先读取配置文件、平台规则和 Learning 文件，再进行内容生成、修改、检查或归档。

## 二、读取顺序

执行任何内容任务前，Agent 必须按顺序读取：

1. `config/creator-profile.template.md` 或用户已复制出的 `config/creator-profile.md`
2. `config/content-pillars.template.md` 或用户已复制出的 `config/content-pillars.md`
3. 对应平台规则文件
4. `Learning.md`
5. 相关历史草稿、最终稿、复盘、样本、脚本和预测文件

如果用户尚未完成配置，Agent 应先引导用户完成配置，而不是直接开始生成内容。

## 三、选题前规则

推荐任何选题前，Agent 必须完成：

1. 创作者定位匹配检查。
2. 内容主线匹配检查。
3. 平台适配检查。
4. 历史查重。
5. 风险检查。
6. 差异化判断。

Agent 不得直接凭当前灵感推荐选题。

查重时至少检查：

- 当前平台 `drafts/`
- 当前平台 `final/`
- 当前平台 `retros/`
- 当前平台 `samples/`
- 当前平台 `scripts/`
- 当前平台 `predictions/`
- 其他平台相关内容
- 全局样本、脚本、预测和复盘文件

## 四、写作前规则

写作前必须确认：

1. 目标平台。
2. 内容类型。
3. 当前产物状态。
4. 是否需要事实核验。
5. 是否涉及隐私、版权或专业风险。
6. 是否需要用户补充材料。

如果信息不足以开始写作，Agent 应先向用户说明缺口，并请求补充材料。

## 五、内容钩子规则

图文类内容必须检查：

1. 标题。
2. 首图。
3. 前两行正文。
4. 多图递进。
5. 隐私和版权风险。

视频类内容必须检查：

1. 前三秒口播。
2. 前十秒承接。
3. 节奏推进。
4. 画面辅助。
5. 结尾讨论口。

长文类内容必须检查：

1. 标题。
2. 开篇问题。
3. 结构推进。
4. 依据和边界。
5. 结尾判断。

## 六、视觉制作规则

Agent 不得默认某一种视觉形式适用于所有选题。

制作视觉素材前，必须确认：

1. 内容类型。
2. 目标平台。
3. 用户参考案例。
4. 视觉形式。
5. 是否使用真实截图。
6. 是否使用信息卡、流程图、生成图或 HTML 幻灯片。
7. 是否存在隐私风险。
8. 是否需要第三方 Skill。

## 七、第三方 Skill 使用规则

第三方 Skill 应作为内容生产链条中的专业能力模块使用。

- 内容评分、发布前预测、发布后复盘可使用 `cheat-on-content`。
- 图文图片卡片、知识卡、流程图可使用 `baoyu-skills / baoyu-xhs-images`。
- HTML 幻灯片、视频辅助画面、复杂概念可视化可使用 `frontend-slides`。

调用第三方 Skill 前，Agent 应先判断该 Skill 是否适合当前任务。

同步或修改第三方 Skill 前，必须核对许可证，保留原作者信息和原 LICENSE。

## 八、定稿规则

所有可能发布的内容必须经过用户明确确认。

用户未确认前，只能放入 `drafts/` 或 `中间产物/`，不得放入 `final/` 或 `最终稿/`。

Agent 在内容接近可发布状态时，必须询问：

```text
这是否确认为最终稿？
```

## 九、Learning 更新规则

复盘后，如出现稳定偏好、用户纠正、有效经验、失败判断或被推翻假设，Agent 应询问用户是否写入 `Learning.md`。

Agent 不得擅自修改长期规则。

写入 Learning 时，应记录：

1. 日期。
2. 经验类型。
3. 具体规则。
4. 适用范围。
5. 来源或证据。

## 十、隐私规则

Agent 不得写入或公开：

- 真实姓名
- 真实账号
- 电话
- 邮箱
- 地址
- 客户信息
- 案号
- 合同
- 聊天记录
- 平台后台数据
- token
- cookie
- secret
- auth 信息

所有真实信息必须使用占位符。

常用占位符包括：

```text
<创作者名称>
<平台A>
<平台B>
<平台C>
<内容主线A>
<目标受众>
<项目A>
<SkillA>
<隐私信息>
```
